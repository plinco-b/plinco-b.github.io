Plinco — Complete App State

Every workflow, iOS & Android, accurate to the current build v0.25.3

How to read this deck: the phones below are real screenshots captured from a live iOS Simulator (iPhone 15, iOS 26.5) and Android Emulator (Pixel 9a) running this exact codebase via dev-client builds — not mockups. Each is tagged REAL.

A handful of screens genuinely require two devices simultaneously paired in the same village — wolf-in-the-way, the on-call button while actually on a call, mute/speaker, the distress-escalation check-in modal, the parent-side battery/thermal chip, banish, and overthrow. Two real app bugs found while building this deck (same-login auto-join doesn't actually join the existing village; the pairing-code generator hangs on dashes — see closing notes) blocked getting two simulators into one live session in this environment. Those screens fall back to hand-built CSS mockups using the real theme tokens and exact current copy, each tagged MOCKUP — LIVE PAIRING BLOCKED.

A smaller number of transient states (mid-cry-detection distress, the wolf-free "all quiet" sleepy sheep animation mid-frame, etc.) weren't feasible to trigger without real audio or a timed event in this pass — these are tagged MOCKUP — NOT CAPTURED.

Sections marked CONCEPT are not built yet — original interaction designs for review, not existing screens.

Honesty notes for this deck
What's real vs. mockup vs. concept: every REAL screen above is an actual screenshot captured from a running iOS Simulator (iPhone 15) or Android Emulator (Pixel 9a) dev-client build of this exact codebase. MOCKUP — LIVE PAIRING BLOCKED screens are hand-built from the real theme tokens and exact current copy (file:line cited) but genuinely require two devices paired in one village — see the bugs below. MOCKUP — NOT CAPTURED screens are real, reachable states this session simply didn't trigger (needs live cry audio, a network kill, etc.) — also copy-verified against source. Sections ⑦–⑧ marked CONCEPT are original design work for review, not implemented.

Three real app bugs found while building this deck:
1. Same-login auto-join doesn't work. Signing into the same account on a second device does not join the existing village — each new device becomes chief of its own separate 0/3 village, contradicting the "Login with {email} to automatically connect" copy on the invite card.
2. Pairing-code generation hangs. The manual invite-code field (generatePairingCode() in src/pairing/deviceLink.ts) never resolves past placeholder dashes on either device tested.
3. Fresh-signup FTUE skips the welcome basket. New signups route straight to baby setup, bypassing the dashboard and the PuppyDropDiscovery reveal animation entirely — the dashboard afterward also shows "0" puppies, suggesting the welcome-basket grant isn't firing or isn't visible in this environment.

Pre-existing logging gap: src/analytics/index.ts's typed event registry covers cry/wake/answer/session-lifecycle events but has no event for sign-up/sign-in failures or the consent-checkbox-blocked tap — those Alerts are local-only today, nothing leaves the device.
